# Telegram Web App — Калькулятор заработка

Полноценный веб-аналог Telegram бота. Работает с той же Google Sheets таблицей,
использует те же API ключи, ту же логику ротации и получения реферальных ссылок.
Скриншоты отправляются в канал модерации через Telegram Bot API.

## Архитектура

```
frontend/app.jsx    →  React (Telegram Web App)
backend/server.py   →  Flask API → Google Sheets + Telegram Bot API
```

## Что реализовано

### Из Google Sheets (как в bot.py):
- ✅ Офферы из «Список офферов X» и «Список офферов Y»
- ✅ Тексты из листа «Тексты» (с подстановкой переменных)
- ✅ Настройки из листа «Настройки» (картинки, ссылки кураторов)
- ✅ API ключи из «API» и «Дополнительный АПИ»
- ✅ Ротация по листу «Ротация»
- ✅ Реферальные ссылки из «ССЫЛКИ» + API Rafinad (fallback)
- ✅ Запись заявок в «Юзеры» (телефон, возраст, ссылка)
- ✅ Чтение заявок и статусов из «Юзеры»
- ✅ Рефералы → лист «Рефералы»

### Отправка в канал модерации:
- ✅ Скриншоты оформления → канал с кнопками ✅/❌
- ✅ Фото получения карты → канал с кнопками ✅/❌
- ✅ Caption с ID, username, телефоном, картой, хэштегом #uXXX

### Пользовательская логика:
- ✅ Выбор возраста (запоминается навсегда)
- ✅ Выбор карт (чекбоксы)
- ✅ Расчёт заработка с бонусами (+10% за 2, +15% за 3+)
- ✅ Запрос телефона (один раз)
- ✅ Получение реферальных ссылок через API Rafinad (синхронно)
- ✅ Отправка скриншотов
- ✅ Личный кабинет с актуальными статусами
- ✅ Подтверждение получения карты
- ✅ Повторная отправка скриншота
- ✅ Вывод средств (ссылка на куратора)
- ✅ Реферальная программа (генерация ссылок)
- ✅ Оформить ещё карты (фильтрация оформленных)

### Чего НЕТ на клиенте (и не должно быть):
- ❌ Кнопки модерации (одобрить/отклонить) — только в канале
- ❌ Прямой доступ к Google Sheets — всё через бэкенд
- ❌ Моковые/рандомные данные

## Установка и запуск

### 1. Бэкенд

```bash
cd backend
pip install -r requirements.txt

# Положить JSON ключ рядом с server.py
cp /path/to/debet-485119-31d092561d4c.json .

# Запустить
python server.py
```

Сервер поднимется на `http://localhost:5000`

### 2. Фронтенд

Фронтенд — это React-компонент (`app.jsx`). Для использования как Telegram Web App:

**Вариант A — через Vite (рекомендуется):**

```bash
cd frontend
npm create vite@latest . -- --template react
# Заменить src/App.jsx содержимым app.jsx
npm install
npm run build
# Скопировать dist/ в backend/frontend/dist/
```

**Вариант B — через артефакт Claude.ai:**

Просто скопируйте содержимое `app.jsx` в артефакт — он будет работать
как превью (API_BASE нужно указать на ваш сервер).

### 3. Telegram Web App

В BotFather:
1. `/mybots` → выбрать бота → Bot Settings → Menu Button
2. Указать URL вашего сервера (например через ngrok)

Или добавить inline-кнопку в боте:
```python
types.InlineKeyboardButton("🌐 Web App", web_app=types.WebAppInfo(url="https://your-domain.com"))
```

### 4. Деплой

Рекомендуется Railway / Render / VPS:
```bash
# На сервере
cd backend
pip install -r requirements.txt
gunicorn server:app --bind 0.0.0.0:5000
```

Или Docker:
```dockerfile
FROM python:3.11-slim
WORKDIR /app
COPY backend/ .
COPY frontend/dist/ ../frontend/dist/
RUN pip install -r requirements.txt
CMD ["gunicorn", "server:app", "--bind", "0.0.0.0:5000"]
```

## API Endpoints

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/api/init` | Инициализация: настройки, наличие заявок, возраст, телефон |
| POST | `/api/offers` | Загрузить офферы по возрасту |
| POST | `/api/orders` | Получить заявки пользователя |
| POST | `/api/get_ref_link` | Получить реферальную ссылку на оффер |
| POST | `/api/submit_screenshot` | Отправить скриншот (→ Google Sheets + канал) |
| POST | `/api/submit_receipt` | Отправить фото карты (→ канал) |
| POST | `/api/resubmit_screenshot` | Повторная отправка скриншота |
| POST | `/api/check_status` | Проверить актуальный статус строки |
| POST | `/api/referral/create` | Создать реферальную ссылку |
| GET  | `/api/texts` | Все тексты и настройки |

## Совместимость с bot.py

Веб-апп и бот работают с одной и той же таблицей Google Sheets.
Заявки, созданные через веб-апп, видны боту и наоборот.
Модерация (одобрение/отклонение) работает через того же бота в канале.
